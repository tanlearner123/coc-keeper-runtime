## Context

The existing test infrastructure (`tests/fakes/`) can fake Discord interactions and model calls, but cannot drive full runtime scenarios without Discord. Existing tests (E60-E68) validate layer-by-layer contracts but cannot answer "can the system run a full session from start to finish without live Discord?"

This phase creates the foundational driver layer that all subsequent phases (E70-E72) depend on.

**Key constraints from research:**
- `BotCommands` can be called directly with `FakeInteraction` ŌĆö VERIFIED
- `TurnCoordinator.handle_turn()` / `stream_turn()` are good entry points
- `SessionStore.dump_sessions()` / `load_sessions()` provide state serialization
- `RulesEngine` accepts `dice_roller=` injection ŌĆö just need seeded implementation
- `rules/dice.py` uses `random.randint` directly ŌĆö MUST add seeded variant to source

### Task 4: Create RuntimeTestDriver

**What:** Create `src/dm_bot/testing/runtime_driver.py` ŌĆö unified Discord-free runtime interface.

**Why:** All scenario steps (command, message, state query) must go through one interface that doesn't require Discord.

**Interface:**
```python
class RuntimeTestDriver:
    def __init__(
        self,
        *,
        dice_seed: int | None = None,
        model_client: StubModelClient | None = None,
        db_path: str = ":memory:",
        clock: FakeClock | None = None,
    ): ...

    async def start(self) -> None:
        """Initialize runtime, start services."""

    async def stop(self) -> None:
        """Shutdown runtime."""

    async def run_command(
        self,
        actor_id: str,
        command: str,  # e.g. "bind_campaign"
        args: dict,
    ) -> StepResult:
        """Run a slash-command-like action."""

    async def send_message(
        self,
        actor_id: str,
        text: str,
        channel: str | None = None,
    ) -> StepResult:
        """Send a natural message as if from a player."""

    def snapshot_state(self) -> dict:
        """Return current session state snapshot."""

    def snapshot_db(self) -> dict:
        """Return current DB snapshot."""

    def get_outputs(self, audience: str) -> list[OutputRecord]:
        """Get emitted outputs for a specific audience (kp, public, player_p1, etc)."""

    def get_phase(self) -> str:
        """Get current session phase."""

    def get_phase_history(self) -> list[dict]:
        """Get phase transition history."""

    async def restart_runtime(self) -> None:
        """Simulate runtime restart ŌĆö reload state from snapshot."""

    async def simulate_crash(self) -> None:
        """Simulate mid-session crash."""

    async def simulate_stream_interrupt(self) -> None:
        """Simulate streaming output interruption."""
```

**StepResult shape:**
```python
@dataclass
class StepResult:
    phase_before: str
    phase_after: str
    emitted_outputs: list[OutputRecord]
    state_diff: dict  # {field: (old, new)}
    persistence_events: list[dict]
    error: str | None = None
    duration_ms: float = 0.0
```

**OutputRecord shape:**
```python
@dataclass
class OutputRecord:
    audience: str  # "public", "kp", "player_p1", "player_p2"
    content: str
    timestamp: float
    message_type: str  # "narration", "system", "dice_result", "error"
```

**How:**

2. Read `src/dm_bot/orchestrator/turns.py` ŌĆö understand `TurnCoordinator` entry points
3. Read `src/dm_bot/orchestrator/gameplay.py` ŌĆö understand `GameplayOrchestrator`
4. Implement `RuntimeTestDriver.__init__`:
   - Creates `PersistenceStore(db_path)` or `:memory:`

**Acceptance:**

### Task 5: Create ScenarioRunner

**What:** Create `src/dm_bot/testing/scenario_runner.py` ŌĆö executes YAML scenario scripts.

**Why:** Scenarios are defined as YAML files. Runner parses them, executes steps via `RuntimeTestDriver`, collects results, writes artifacts.

**Interface:**
```python
class ScenarioRunner:
    def __init__(self, driver: RuntimeTestDriver): ...

    async def run(
        self,
        scenario_path: str | Path,
        *,
        write_artifacts: bool = True,
        artifact_dir: str | Path = "artifacts/scenarios",
        fail_fast: bool = False,
    ) -> ScenarioResult: ...

@dataclass
class ScenarioResult:
    scenario_id: str
    passed: bool
    steps: list[StepResult]
    phase_timeline: list[str]
    final_state: dict
    failure: Failure | None
    artifact_dir: Path | None
```

**How:**
1. Parse YAML scenario file (actors, fixtures, steps, assertions)
2. Initialize `RuntimeTestDriver` from fixture config (dice_seed, model_mode, db_mode)
3. Execute steps in order via driver
4. Collect `StepResult` for each step
5. After all steps, run assertions
6. Write artifacts (JSON timeline, outputs by audience, state snapshots)
7. If any assertion fails, classify with `FailureCode` and write `failure.json`

**Acceptance:**
- `runner.run("tests/scenarios/acceptance/scen_happy_path.yaml")` returns `ScenarioResult`
- `passed=True` when all assertions pass
- `passed=False` with `FailureCode.PHASE_TRANSITION_MISMATCH` when phase wrong
- Artifacts written to `artifacts/scenarios/<scenario-id>/`
