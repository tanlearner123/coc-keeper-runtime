## Domain

Add concise player-facing explanations for why messages were ignored, buffered, deferred, or otherwise routed differently. This completes the player-facing visibility work started in Phases 51-52 by actually surfacing the handling reasons that were already captured in the `VisibilitySnapshot`.

**Core value:** Players should never wonder "why was my message ignored?" — they should receive short practical explanations at the right moments.

## Decisions

### D-01: Feedback Delivery Mechanism
**Decision:** Send routing feedback as ephemeral Discord messages (visible only to the sender).

**Rationale:**
- Other players don't need to know why a specific player's message was handled differently
- Ephemeral messages are the standard Discord pattern for player-specific feedback
- Keeps game channel clean while informing the player

### D-02: Feedback Timing
**Decision:** Feedback is delivered immediately when a message is buffered, ignored, or deferred — not delayed until scene resolution.

**Rationale:**
- Players need immediate confirmation their message was received
- Delayed feedback causes confusion and duplicate messages
- Buffered messages will also show feedback when buffered

### D-03: Conciseness Standard
**Decision:** Feedback must be ≤50 characters for ordinary routing (buffered/ignored), with optional extended explanation available via command.

**Rationale:**
- PLAY-04 requires conciseness for ordinary play channels
- 50 characters fits in a single Discord embed line
- Command-based detail retrieval keeps game channel uncluttered

### D-04: Feedback Content by Routing Outcome
| Outcome | Feedback Type | Example |
|---------|--------------|---------|
| BUFFERED | Short reason + when released | "⏳ 行动已收集，结算后显示" |
| IGNORED | Short reason | "⏸️ 消息在当前阶段不适用" |
| DEFERRED | Short reason | "📋 规则问题将在结算后回答" |
| PROCESSED | No feedback (action is handled) | N/A |

## Specifics

### PLAY-03: Short Practical Explanations

**Target scenarios:**
1. Player submits action during SCENE_ROUND_RESOLVING → buffered
2. Player types OOC during SCENE_ROUND_OPEN → buffered with guidance
3. Player asks rules question during COMBAT → deferred
4. Player tries action during AWAITING_READY → ignored or buffered
5. Unknown intent → ignored with guidance

**Feedback message structure:**
```
[emoji] [short explanation] ([optional follow-up])
```

Examples:
- "⏳ 行动已记录，结算后显示"
- "📋 OOC消息将在回合结束后显示"
- "⏸️ 战斗中无法使用规则查询"

### PLAY-04: Concise for Play Channels

**Constraints:**
- Primary feedback: ≤50 characters visible in ephemeral
- Detailed feedback: Available via `/status` or `/help routing`
- No feedback spam: Only one feedback per message, not per system check
